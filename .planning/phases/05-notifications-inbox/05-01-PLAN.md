## Objective

Thêm **lớp persistence + domain** cho inbox trên Mongo social: schema, index, repository, service (list có cursor + `unreadOnly`, mark read, retention, filter blocked trên list — D-14).

## Scope

- In: `notification.schema.ts`, `database.service.ts` (getter + `createNotificationIndexes` trong `initializeIndexes`), `notification.repository.ts`, `notification-cursor.util.ts` (encode/decode giống pattern `chat-cursor.util.ts` với keys `t` + `i`), `notifications.service.ts`.
- Out: HTTP routes (plan 02); Socket emit + hooks friends/chat (plan 03) — service có thể nhận `onCreated?: (recipientId, dto) => void` hoặc inject `SocketService` rỗng tới khi plan 03 wire.

## Suggested document shape

- `_id`, `recipientId: ObjectId`, `read: boolean`, `readAt?: Date`, `createdAt: Date`, `type`, `actor: { userId: string, displayName: string, avatar?: string }`, `payload` object — **TypeScript discriminated union** theo `type`:
  - `friend_request`: `{ fromUserId: string }` (hoặc chỉ dựa actor — tối thiểu cần id điều hướng; align OpenAPI sau)
  - `friend_accepted`: `{ friendUserId: string }`
  - `new_message`: `{ conversationId: string, messageId: string, previewText?: string, previewKind: 'text' | 'attachment' | 'mixed' }` — D-06: chỉ `previewText` khi text thuần; chỉ file → `previewKind: 'attachment'`, không nhét nội dung file
  - `added_to_group`: `{ conversationId: string, chatName?: string }`

**Index đề xuất:** `{ recipientId: 1, createdAt: -1, _id: -1 }` cho list; optional `{ recipientId: 1, read: 1, createdAt: 1 }` hỗ trợ trim (hoặc dùng sort trong query trim).
