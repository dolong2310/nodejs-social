## Objective

Định nghĩa **schema + index Mongo** trên **chat database** và wire **bootstrap index** khi app start — chưa expose HTTP.

## Scope

- In: new schema files (`IChat`, `IChatMember`, `IChatMessage` hoặc tên tương đương), `DatabaseService` getters, `initializeChatIndexes()`, gọi từ `createApp` / `DatabaseInstance` sau `connect` (cùng chỗ với social indexes hoặc ngay sau).
- Out: REST routes, business rules friends/block (plan sau).

## Suggested document shapes (executor refine)

- **`chats`:** `_id`, `type: 'direct' | 'group'`, `createdBy`, `createdAt`, `updatedAt`, `name?`, `avatarMediaId?`, direct-only fields: `userIdLow`, `userIdHigh` (sorted pair) **unique**; group: không cần pair.
- **`chatMembers`:** `chatId`, `userId`, `role`, `joinedAt`, `lastReadMessageId?`, `lastReadAt?` — unique `(chatId, userId)`.
- **`messages`:** `chatId`, `senderId`, `text?`, `attachments?` (array metadata: key, mime, size, url hoặc s3 key), `createdAt` — index `{ chatId: 1, createdAt: -1, _id: -1 }` cho cursor.
