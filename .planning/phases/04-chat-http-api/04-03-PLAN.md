## Objective

Repositories + **REST API** cho **conversation shell**: direct get-or-create, group create, list conversations của user, chi tiết chat, **members**: invite, leave, kick, đổi role, transfer admin, PATCH name/avatar (admin/manager).

## Scope

- In: `ChatRepository`, `ChatMemberRepository`, (optional stub `ChatMessageRepository` nếu cần cho lastMessage preview), `ChatsService`, controller, validation, DTO, routes, container, `app.ts` mount.
- Out: POST message body đầy đủ, cursor history, mark-read, S3 attachment (04-04, 04-05).

## REST sketch (executor chi tiết OpenAPI)

- `POST /api/.../direct` — `{ peerUserId }` → get/create direct.
- `POST /api/.../groups` — `{ name?, memberIds: string[] }` (≥1 friend + creator implicit).
- `GET /api/...` — list (cursor) các chat user tham gia.
- `GET /api/.../:chatId` — metadata + members summary.
- `PATCH /api/.../:chatId` — name, avatar (admin/manager).
- `POST /api/.../:chatId/members` — `{ userId }` invite.
- `DELETE /api/.../:chatId/members/me` — leave.
- `DELETE /api/.../:chatId/members/:userId` — kick (role guard).
- `PATCH /api/.../:chatId/members/:userId/role` — admin: promote/demote manager↔member.
- `POST /api/.../:chatId/admin/transfer` — `{ newAdminUserId }`.
