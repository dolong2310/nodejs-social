## Objective

Unify **engagement permissions** (like, bookmark, comment-as-post, and view gates) with **verification tier** (D-04, D-05) and **BLCK-02** on **all read and mutation paths** not yet covered by plan `02` alone (single post, comments list, bookmarks, likes). Add **likes** persistence + HTTP (minimal mirror of bookmarks). Fix **guest / unverified read** wiring so `optionalAuth` no longer runs `userVerifiedValidation` on GET posts.

## Scope

- In: auth/validation middleware, `posts.validation` / `posts.route`, `posts.controller` `getUserId` optional for views, `bookmarks` pipeline, new likes stack, `message.constant` 403 strings, `DatabaseService` `likes` collection + index, `app.ts` `/likes`, `Container`.
- Out: Swagger polish (plan `04`), search, feed query construction (plan `02`).
